## Update 2026-07-09: Внедрение защитных механизмов и аудит безопасности (SECURITY_AUDIT)

В рамках аудита безопасности проекта `LawTrack` на основе OWASP Top 10:2025 и регламента безопасности были реализованы следующие улучшения:
1. **Предотвращение утечки данных (Mishandling of Exceptional Conditions)**: Обновлен класс `GlobalExceptionHandler`. Теперь при возникновении непредвиденных ошибок (`Exception`) стек-трейсы и детальные сообщения об ошибках логируются на стороне сервера (через Slf4j), а клиенту возвращается безопасное нейтральное сообщение на русском языке ("Произошла внутренняя ошибка сервера").
2. **Защита от XSS (Stored XSS)**: Создан утилитарный класс `HtmlSanitizer` в пакете `com.lawtrack.util`, который автоматически очищает текстовые поля ввода (имя клиента, описание дела, текст заметок) от HTML-тегов перед их сохранением в БД.
3. **Безопасность CORS (Security Misconfiguration)**: Реализован класс `CorsConfig` в пакете `com.lawtrack.config`. Он считывает список разрешенных доменов из переменной окружения `cors.allowed-origins` (с локальным фоллбэком) и гибко настраивает CORS без использования опасной маски `*` при включенных куках (`allowCredentials`).
4. **Внедрение HTTP-заголовков безопасности (Security Headers)**: Создан сервлет-фильтр `SecurityHeadersFilter`. Он инжектирует в заголовки всех HTTP-ответов политики `Content-Security-Policy`, `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`, `Referrer-Policy` и `Permissions-Policy`.
5. **Предотвращение DoS и брутфорса (Rate Limiting)**: Создан потокобезопасный in-memory фильтр `RateLimitingFilter`, который ограничивает количество запросов с одного IP-адреса до 120 запросов в минуту.
6. **Защита доступа к API (Broken Access Control)**: Создан фильтр `AdminTokenFilter`, реализующий проверку опционального токена администратора (`X-Admin-Token` в заголовке). Для устойчивости к Timing-атакам сравнение токенов осуществляется с помощью константного времени через `MessageDigest.isEqual`.
7. **Конфигурация Actuator**: В `application.yml` явно ограничены открытые эндпоинты Actuator (разрешен только `/actuator/health`).
8. **Обновление фронтенда**: API-клиент фронтенда `api.ts` дополнен логикой автоматической отправки заголовка `X-Admin-Token`, если в окружении настроена переменная `NEXT_PUBLIC_ADMIN_TOKEN`.
9. **Верификация**: Проект успешно скомпилирован и прошел все тесты, сборка Next.js собрана без предупреждений, скрипт `verify-all.ps1` завершился успешно.
