## Точная реализация (Technical Design)

### React Query Hooks
1. `use-clients.ts`: Обертка над `useQuery` для `fetchClients` с ключом `["clients", filters]`.
2. `use-create-client.ts`: Обертка над `useMutation` для `createClient`. При успехе инвалидирует ключи `["clients"]` и `["stats"]`.
3. `use-update-status.ts`: Обертка над `useMutation` для `updateClientStatus(id, status)` с реализацией **Optimistic Updates**:
   - `onMutate`: отменить текущие запросы `["clients"]`, сохранить предыдущее состояние, обновить локальный кэш React Query измененным статусом клиента.
   - `onError`: откатить кэш назад к сохраненному состоянию.
   - `onSettled`: инвалидировать `["clients"]` и `["stats"]` для синхронизации с сервером.

### Form Validation (Zod Schema)
```typescript
const clientSchema = z.object({
  name: z.string().min(2, "Введите имя").max(150),
  phone: z.string().regex(/^\+?[0-9\s\-()]{7,20}$/, "Некорректный номер телефона"),
  caseDescription: z.string().optional(),
  deadline: z.string().optional(),
});
```

### UI Components
- `ClientTable`: Рендерит заголовки колонок (Имя, Телефон, Статус, Дата добавления, Действия) и список строк.
- `ClientRow`: Рендерит ячейки. В ячейке "Статус" отображает бейдж статуса. При клике на бейдж открывается выпадающий список (выбор из трех статусов). При выборе запускается мутация.
- `AddClientDialog`: Вызывается по клику на "+ Добавить клиента". Содержит форму с полями: ФИО, Телефон, Описание дела (caseDescription), Срок решения (deadline, тип `date`).

## Риски и подводные камни (Edge Cases)

- **Откат Optimistic Update:** Если бэкенд возвращает ошибку (например, неверный формат телефона или ошибка БД), фронтенд должен корректно вернуть статус в таблице к исходному состоянию и показать тост с ошибкой (через sonner или обычный alert).
- **Сброс формы:** Очищать поля формы после успешного добавления клиента и закрытия модалки.

2. Нажать кнопку "+ Добавить клиента". Заполнить форму корректными и некорректными данными (проверить валидацию). Сохранить корректного клиента.

4. Кликнуть на статус клиента в таблице, перевести его во "В работе" (IN_PROGRESS). Убедиться, что статус обновился мгновенно, счетчики обновились, и пришло уведомление в Telegram.
