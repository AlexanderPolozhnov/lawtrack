# LawTrack — Мини-CRM для юриста ⚖️

**LawTrack** — это современная легковесная CRM-система, созданная специально для юристов и небольших юридических практик. Она позволяет вести учет клиентов, отслеживать статусы дел, контролировать дедлайны, вести историю событий по каждому делу и автоматически отправлять уведомления в Telegram при важных изменениях.

Проект разработан в рамках демонстрации возможностей современной Full-Stack разработки с использованием автономии ИИ-агентов.

---

## 🛠️ Стек технологий и обоснование

При проектировании архитектуры мы сознательно выбрали следующий технологический стек:

*   **Backend:**
    *   **Java 21** и **Spring Boot 4.0** — стандарт де-факто для надежных корпоративных приложений с отличной производительностью и встроенной поддержкой современных стандартов Java.
    *   **Spring Data JPA / Hibernate** — для надежного и быстрого доступа к данным.
    *   **Flyway** — для строгого эволюционного контроля схемы базы данных через миграции.
    *   **Jakarta Bean Validation** — для обеспечения надежной валидации входных данных (DTO) на уровне API.
    *   **MapStruct** — для эффективного и безопасного маппинга между сущностями базы данных и DTO без написания рутинного кода.
    *   **Spring Boot Actuator** — для отслеживания работоспособности сервиса (health checks) при деплое в облако.
*   **Frontend:**
    *   **Next.js 16 (App Router)** и **React 19** — передовой стек для сборки быстрых и отзывчивых веб-интерфейсов.
    *   **TypeScript** — для статической типизации и минимизации ошибок на этапе разработки.
    *   **Tailwind CSS v4** — для построения современной, отзывчивой и красивой дизайн-системы с использованием CSS-переменных и плавных микроанимаций.
    *   **shadcn/ui** — библиотека доступных и стильных компонентов (таблица, диалоги, выпадающие списки, боковые панели).
    *   **TanStack Query v5 (React Query)** — для умного кэширования данных, автоматической инвалидации кэша и реализации **Optimistic Updates** (оптимистичное обновление интерфейса при смене статусов).
    *   **React Hook Form + Zod** — для удобной валидации форм на стороне клиента.
*   **Инфраструктура:**
    *   **PostgreSQL 16** — основная реляционная СУБД для продакшена.
    *   **H2 (In-Memory)** — для сверхбыстрого запуска интеграционных тестов и локальной разработки без зависимостей.
    *   **Docker & Docker Compose** — для контейнеризации и быстрого локального развертывания.

> [!NOTE]
> **Почему выбран именно такой стек?**
> Взят классический стек, используемый в реальных продакшн-проектах (Spring Boot 4 + Next.js), чтобы продемонстрировать не просто учебный пример, а промышленный подход к организации монорепозитория. Архитектура не перегружена лишними звеньями (такими как брокеры сообщений или распределенные кэши) ради высокой скорости разработки и простоты развертывания MVP.

## 🚀 Быстрый локальный запуск

Для запуска всего проекта локально выполните следующие шаги.

### Шаг 1. Клонирование и настройка окружения
Создайте файл `.env` в корневом каталоге проекта по шаблону `.env.example`:
```env
# Настройки Telegram (опционально, для уведомлений)
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

# Ссылка на фронтенд для настройки CORS
FRONTEND_URL=http://localhost:3000
```

### Шаг 2. Запуск базы данных в Docker
Запустите PostgreSQL локально с помощью Docker Compose:
```bash
docker compose up -d
```
Это поднимет базу данных PostgreSQL 16 с параметрами подключения из `docker-compose.yml`.

### Шаг 3. Запуск Backend (Spring Boot)
Перейдите в директорию `backend` и запустите приложение:
```bash
cd backend
./mvnw.cmd spring-boot:run
```
*(На Linux/macOS используйте `./mvnw spring-boot:run`)*.  
Бэкенд будет доступен по адресу: `http://localhost:8080`.  
Swagger-документация API: `http://localhost:8080/swagger-ui/index.html`.

### Шаг 4. Запуск Frontend (Next.js)
Откройте новый терминал в корне проекта, перейдите в папку `frontend` и запустите сервер разработки:
```bash
cd frontend
pnpm install
pnpm run dev
```
Фронтенд будет доступен по адресу: `http://localhost:3000`.

## ☁️ Инструкция по деплою в облако

Проект полностью подготовлен для легкого развертывания на бесплатных тарифах популярных облачных провайдеров.

### 1. Деплой Backend (Render.com)
Render идеально подходит для хостинга Spring Boot приложений в Docker-контейнерах:
1. Зарегистрируйтесь на [Render.com](https://render.com/) и создайте базу данных **PostgreSQL** (Free tier). Скопируйте строку подключения (External Database URL).
2. Нажмите **New** -> **Web Service**. Подключите ваш GitHub репозиторий.
3. Укажите имя сервиса и выберите **Runtime: Docker**. Render автоматически найдет [backend/Dockerfile](file:///C:/.development/Projects/LawTrack/backend/Dockerfile).
4. Задайте **Root Directory** как `backend` (если деплоите из монорепозитория).
5. Перейдите во вкладку **Environment** и добавьте следующие переменные окружения:
   *   `DB_HOST`: Хост вашей базы данных Render (например, `dpg-xxx-a.render.com`).
   *   `DB_NAME`: Имя базы данных (`lawtrack`).
   *   `DB_USER`: Имя пользователя базы данных.
   *   `DB_PASSWORD`: Пароль от базы данных.
   *   `TELEGRAM_BOT_TOKEN`: Токен вашего бота.
   *   `TELEGRAM_CHAT_ID`: ID чата в Telegram.
   *   `FRONTEND_URL`: URL задеплоенного фронтенда на Vercel (например, `https://lawtrack.vercel.app`).
   *   `PORT`: `8080`.
6. Нажмите **Deploy Web Service**.

> [!WARNING]
> **Слип-мод Render (Free Tier):**
> Поскольку используется бесплатный тариф Render.com, контейнер бэкенда засыпает после 15 минут отсутствия активности. Первый запрос после сна (например, открытие Swagger или загрузка списка клиентов) может занять 30-50 секунд. Пожалуйста, дождитесь инициализации. Мы использовали минималистичный Docker-образ на базе Alpine, чтобы ускорить время холодного старта.

### 2. Деплой Frontend (Vercel)
Vercel предоставляет нативную интеграцию с Next.js:
1. Зарегистрируйтесь на [Vercel.com](https://vercel.com/) и нажмите **Add New** -> **Project**.
2. Импортируйте ваш репозиторий с монорепозиторием.
3. В настройках проекта укажите **Root Directory** как `frontend`.
4. В разделе **Environment Variables** добавьте:
   *   `NEXT_PUBLIC_API_URL`: URL вашего бэкенда на Render (например, `https://lawtrack-backend.onrender.com`).
5. Нажмите **Deploy**. Vercel автоматически соберет проект и предоставит рабочий URL.

### 3. Настройка Telegram-уведомлений
Для работы уведомлений:
1. Напишите боту `@BotFather` в Telegram, отправьте команду `/newbot` и скопируйте полученный API-токен.
2. Напишите вашему созданному боту `/start`.
3. Чтобы узнать свой `chat_id`, перейдите по адресу:
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
   и найдите поле `"chat":{"id":...}` в ответе.
4. Пропишите эти значения в настройки бэкенда. При добавлении клиентов или смене статуса вам будут приходить красивые асинхронные уведомления в Telegram.

## 🔒 Контроль качества (Скрипты проверки)

Перед отправкой изменений в репозиторий мы всегда запускаем полный цикл тестов и линтинга.
Скрипт верификации:
```powershell
.\verify-all.ps1
```
Он компилирует Java-код, прогоняет тесты Spring Boot, проверяет TypeScript типы и собирает продакшн-билд Next.js.
Если сборка не успешна, пуш блокируется Git-хуками.
